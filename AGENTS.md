# リポジトリ運用ガイド

## 基本方針
- ユーザー（人間）の要求と、実装に必要な要件を分離して管理します。

## ドキュメント定義

### `human-requirement.md`
- ユーザー（人間）の指示・要求を整理して記載するファイルです。
- **拡大解釈をしない**こと。
- ユーザーが言っていない要件や仕様を追加で書かないこと。

### `AI-implement-requirement.md`
- `human-requirement.md` を実現するために必要な内容を記載するファイルです。
- 実装方針、実現手段、技術的な補足事項をここにまとめます。

## 実装フロー

1. ユーザー（人間）の要求を整理する。
2. `human-requirement.md` にその要求を記載する（拡大解釈しない）。
3. `AI-implement-requirement.md` に実現に必要な内容を記載する。
4. `AI-implement-requirement.md` に基づいて実装する。
